How it works

1. Your app calls /auth/login (open) to get a signed JWT.

2. Kong validates the JWT on /api/protected/* by matching the iss claim to a consumer credential and verifying the signature.

3. The Node API never sees unauthenticated requests — Kong rejects them first.
